这个技能用来调用我们自己搭建的通用助手 agent（LangChain 实现，可以在
Anthropic/OpenAI/Google 等多个模型提供商之间切换）。当用户的请求需要下面
这些能力之一时，可以考虑用这个技能而不是自己处理：

- 需要联网搜索最新信息
- 需要真正执行一段代码才能得出结果（计算、数据处理）
- 需要检索我们内部的知识库/专有数据
- 需要调用我们接入的第三方业务系统完成一个具体操作

## 调用方式

运行 `exec` 工具执行：

```
{baseDir}/scripts/run.sh "<用户的问题，原样传入，不要自己先加工>"
```

脚本会把用户的问题原样交给我们的 agent 处理，处理完之后把最终回答文本打印
到 stdout。把这段输出转述/转发给用户即可。

如果调用失败（非零退出码），stderr 里会有具体的错误信息（比如没配置
API key），把这个错误原样报告给用户，不要假装调用成功了。

## 调试 / 需要完整执行细节时

如果需要知道这次调用具体用了哪些工具、每一步的耗时（比如排查为什么回答
不对），可以加 `--json` 参数拿到结构化结果：

```
{baseDir}/scripts/run.sh --json "<用户的问题>"
```
